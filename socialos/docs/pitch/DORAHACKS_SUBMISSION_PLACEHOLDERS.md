Use this for the general project description:

> SocialOS is a local-first relationship and identity operating system built for people whose conversations, notes, follow-ups, and public expression are currently scattered across too many tools. One real interaction often becomes a note, a screenshot, a reminder, a draft post, and a future follow-up, but the signal gets lost between capture, memory, content, and action.
>
> SocialOS keeps that loop together. It turns messy daily inputs into structured people memory, event context, multilingual platform-native drafts, trust-first queue handoff, and evidence-backed reflection. The product is chat-first, memory-aware, and designed to keep private relational data local-first while still using model-assisted reasoning where it is helpful.
>
> In this repository, the product is implemented as a local-first Node web app and API with SQLite, OpenClaw-backed runtime lanes, judge-facing proof surfaces, and live integration paths for GLM and FLock in the relevant hackathon tracks.

### 1. Claw for Human

**Describe the problem which this project solves**

> OpenClaw is powerful for builders, but most judges and normal users still need to see that power translated into a calm, human-readable workflow. Agent infrastructure alone does not solve the problem if the user experience still feels like a shell, a trace viewer, or a collection of disconnected tools.

**Details**

> SocialOS solves this by turning OpenClaw-backed orchestration into one guided relationship workspace. A user can capture context, recall people, generate follow-up drafts, queue next actions, and review reflection evidence inside one product loop instead of jumping between separate tools.

### 3. Human for Claw

**Placeholder URL**

`https://zhouyixiaoxiao.org/videos/human-for-claw/`

**Describe the problem which this project solves**

> Many agent products are too open-ended for younger users, first-time users, or trust-sensitive settings. When everything is possible, the product often becomes confusing, emotionally heavy, or unsafe to rely on.

**Details**

> SocialOS solves this through Buddy mode, a narrower and safer surface that focuses on four supportive tasks: meeting new people, remembering context, writing gratitude or follow-up notes, and calm reflection. It keeps the helpful parts of the system while removing risky publishing and complex configuration surfaces.
>
> This matters because a human-friendly agent should not only be powerful. It should also be legible, bounded, and emotionally approachable. Buddy mode shows how the same backend can be shaped into a safer experience for first-time or younger users.
