# SocialOS DoraHacks 中文排练提示

这份不是正式英文稿，是帮你顺 5 个独立视频的中文提词。

## 总原则
- 这次不是 1 个总视频加一点点替换，而是 5 个都能单独成立的视频。
- 5 个视频的产品主线完全一样，变的是 bounty framing、partner integration、proof 页面。
- 每个视频都必须讲到五件事：
  - problem
  - solution
  - technical implementation
  - bounty partner / API / infrastructure integration
  - short live demo

## 开头 1 分钟
- 先讲问题：人、上下文、内容、跟进被拆散了。
- 再讲产品定义：`local-first relationship and identity operating system`。
- 记得说一句：`one product, five review angles`。

## 中间产品主线
- 固定讲 `capture -> memory -> drafts -> queue -> mirror`。
- `OpenClaw` 只讲成底层协调能力，不要讲成 agent 炫技。
- 强调 `calm product surface` 和 `trust boundary`。

## bounty 部分怎么讲
- `Claw for Human`：重点是把 Claw 变成真人能用的产品界面。
- `Animoca`：重点是 persistent identity 和 long-horizon memory。
- `Human for Claw`：重点是 Buddy mode 的 4 个安全任务。
- `Z.AI General`：重点是 live GLM 已经在真实多语言工作流里。
- `AI Agents for Good`：重点是 live FLock 做 SDG triage，然后进入真实 follow-up。

## 结尾怎么讲
- 一定打开公网 proof 页面，告诉评审录完后可以复核。
- 结尾固定说：`capture, remember, express, hand off, reflect`。
- 不再讲 fallback；这次 live provider 口径要统一成 live integrated。
